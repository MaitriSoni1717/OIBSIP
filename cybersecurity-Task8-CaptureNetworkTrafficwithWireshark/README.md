# Task 8 - Network Traffic Capture and Analysis Using Wireshark

## Objective

The objective of this task was to capture and analyze network traffic using Wireshark. The captured packets were examined to understand how different network protocols communicate and how data travels across a network.

---

# Tools Used

- Kali Linux
- Wireshark
- Firefox Web Browser
- Linux Terminal

---

# What is Wireshark?

Wireshark is a free and open-source network protocol analyzer used to capture and inspect network packets in real time.

It helps security professionals and network administrators:

- Monitor network traffic
- Troubleshoot connectivity issues
- Analyze protocols
- Detect suspicious activity
- Understand network communications

---

# Why Packet Analysis Matters

Packet analysis helps organizations:

- Detect malicious traffic
- Investigate network incidents
- Troubleshoot performance problems
- Understand protocol behavior
- Improve network security

# DNS Packet Analysis

## Filter Used

```text
dns
```

### Observation

DNS requests and responses were captured between:

- Client: 192.168.0.107
- DNS Server: 192.168.0.1

Example domains observed:

- www.google.com
- detectportal.firefox.com
- safebrowsing.googleapis.com

### Purpose of DNS

DNS (Domain Name System) converts human-readable domain names into IP addresses.

Example:

```text
www.google.com → 142.251.x.x
```

### Security Importance

DNS traffic can reveal:

- Websites being accessed
- Potential malicious domains
- DNS spoofing attempts

---

# HTTP Packet Analysis

## Filter Used

```text
http
```

### Observation

HTTP GET requests and server responses were captured.

Example:

```text
GET /success.txt?ipv4 HTTP/1.1
```

Response:

```text
HTTP/1.1 200 OK
```

### Purpose of HTTP

HTTP (HyperText Transfer Protocol) is used for communication between web browsers and web servers.

### Security Importance

HTTP traffic is unencrypted.

Attackers may intercept:

- Credentials
- Cookies
- Sensitive information

HTTPS should be used whenever possible.

---

# TCP Packet Analysis

## Filter Used

```text
tcp
```

### Observation

TCP communication was observed between:

```text
192.168.0.107
```

and

```text
151.101.209.91
```

The following TCP flags were identified:

- SYN
- SYN ACK
- ACK
- FIN ACK

### TCP Three-Way Handshake

TCP establishes connections using:

1. SYN
2. SYN ACK
3. ACK

This process ensures reliable communication between devices.

### Security Importance

TCP analysis helps identify:

- Open connections
- Connection attempts
- Port scanning activity
- Suspicious network behavior
